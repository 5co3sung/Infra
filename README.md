# Infra

실무에서 경험한 인프라 구축, 운영, 마이그레이션, 장애 대응, 보안 점검 내용을 주제별 문서로 정리한 저장소입니다.

Linux 서버 운영 경험과 클라우드/온프레미스 환경 구축 경험을 바탕으로, 반복적으로 사용하는 설정과 절차를 재사용 가능한 형태로 기록하고 있습니다.

## 소개

이 저장소는 인프라 환경을 구축하고 운영하면서 직접 사용했던 설정, 점검 포인트, 트러블슈팅 경험을 정리한 문서형 저장소입니다.

특히 아래와 같은 경험을 바탕으로 문서를 축적하고 있습니다.

- Linux 기반 서버 운영 및 장애 대응
- 고객사 인프라 구축 및 서비스 안정화 작업
- CentOS 7에서 Rocky Linux 8.10으로의 OS 마이그레이션
- Nginx, MariaDB, PostgreSQL 버전 업데이트 및 운영
- Kubernetes, Helm, Jenkins, ArgoCD 기반 운영 환경 경험
- NAVER Cloud 및 AWS 기반 인프라 운영 경험
- ISMS 대응을 위한 취약점 점검 및 보안 조치
- 정기 점검, 예방 유지보수, 서비스 영향도 검토 및 운영 보고

## 저장소 구성

이 저장소는 `구축가이드` 디렉터리를 중심으로 구성되어 있으며, 주제별 구축 절차와 운영 관련 문서를 정리하고 있습니다.

```text
Infra/
└── 구축가이드/
    ├── 1. Nginx/
    ├── 2. DB/
    ├── 3. DevOps/
    ├── 4. WAS/
    └── 5. AI/
```

## 문서 정리 방향

이 저장소의 문서는 아래 방향으로 정리하고 있습니다.

- 실제 구축/운영 과정에서 사용한 절차 중심 정리
- 반복 작업을 줄이기 위한 재사용 가능한 명령어 기록
- OS, 미들웨어, DevOps, AI 환경별 설정 분리
- 점검 포인트 및 운영 시 주의사항 기록
- 추후 노션, 체크리스트, 스크립트로 확장 가능한 형태 유지

## 활용 방법

1. 필요한 기술 영역의 디렉터리로 이동합니다.
2. 구축 또는 설정 문서를 순서대로 확인합니다.
3. 환경에 맞게 설정값을 조정하여 적용합니다.
4. 반복 작업은 별도 스크립트 또는 체크리스트로 재정리합니다.
5. 장애 대응 또는 점검 작업 시 기준 문서로 활용합니다.

## 대상 환경

- 클라우드 및 온프레미스 혼합 환경

- AI 실행 환경 및 관련 패키지 구성 환경

## 기록 목적

이 저장소는 단순한 명령어 모음이 아니라, 실무에서 경험한 인프라 작업을 다시 활용할 수 있도록 남기는 기술 기록을 목표로 합니다.

앞으로도 아래 내용을 계속 확장할 예정입니다.

- OS별 구축 가이드
- 서비스별 실행 절차
- 점검 체크리스트
- 트러블슈팅 사례
- 운영 표준화 문서
- 설치 및 배포 자동화 스크립트
