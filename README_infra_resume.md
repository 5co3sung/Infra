## 소개

이 저장소는 제가 인프라 엔지니어로 근무하며 경험한 내용을 바탕으로,  
Linux 서버 운영, 미들웨어 설정, 데이터베이스 운영, DevOps 환경, AI 실행 환경 구성까지  
주제별로 정리한 인프라 문서 저장소입니다.

고객사 환경과 사내 시스템을 운영하면서 반복적으로 수행했던 작업,  
장애를 예방하기 위해 점검했던 항목,  
OS 및 미들웨어 업그레이드 과정에서 확인했던 체크포인트를  
나중에도 다시 활용할 수 있도록 문서 중심으로 정리하고 있습니다.

## 이 저장소를 만든 이유

실무에서 인프라 작업은 한 번 하고 끝나는 일이 아니라,  
비슷한 구축과 점검을 여러 환경에서 반복하게 되는 경우가 많았습니다.

특히 다음과 같은 작업들은 항상 정확한 기록이 중요했습니다.

- OS 업그레이드 및 마이그레이션
- Nginx / MariaDB / PostgreSQL 설정 변경
- 서비스 영향도 확인 및 작업 절차 정리
- 정기 점검 및 예방 유지보수
- 클라우드 환경 운영
- Kubernetes 및 CI/CD 환경 구성
- 보안 점검 및 취약점 조치

그래서 이 저장소는 단순한 개인 메모가 아니라,  
**실무 경험을 구조화된 문서로 축적하는 기술 기록 저장소**로 운영하고 있습니다.

## 주요 실무 경험

### 1. 고객사 서버 운영 및 마이그레이션
- 고객사 물리/가상 서버의 EOS 정책에 맞춰 OS 업그레이드 수행
- **CentOS 7에서 Rocky Linux 8.10으로 마이그레이션**
- OS 업그레이드에 맞춘 Nginx, MariaDB, PostgreSQL 버전 업데이트
- 서비스 중단을 최소화하기 위한 이중화 구성 및 검증 수행

### 2. 인프라 운영 및 장애 예방
- Linux 기반 인프라 환경 정기 점검 및 예방 유지보수 수행
- CPU, Memory, Disk, Network 자원 모니터링을 통한 사전 장애 예방
- 긴급 작업 및 정기 점검 시 서비스 영향도 분석과 일정 조율 진행
- 고객 문의 대응 및 작업 결과 리포트 제공

### 3. 사내 시스템 및 클라우드 운영
- 사내 Linux 시스템 운영, 장애 대응, 정기 점검 수행
- NAVER Cloud 기반 인프라 운영
- 고가용성(HA) 환경 유지
- 신규 서비스 런칭을 위한 클라우드 인프라 설계 및 구축

### 4. DevOps / Container 환경 운영
- Kubernetes 기반 컨테이너 환경 운영
- Helm Chart 활용
- CI/CD 파이프라인 구축 및 운영 경험

### 5. 보안 및 인증 대응
- ISMS 인증 대응을 위한 취약점 진단 및 보안 조치 수행
- 운영 환경 점검 시 보안 기준을 함께 고려한 설정 관리

### 6. 구축 및 유지보수 경험
- RFID 서버 및 네트워크 유지보수/구축 경험
- VMware ESXi 기반 가상화 서버 구축
- Oracle 데이터베이스 마이그레이션 수행

## 저장소 구성

현재 저장소는 `구축가이드` 디렉터리를 중심으로 정리하고 있습니다.

- `2. DB`

각 디렉터리에는 설치, 설정, 점검, 운영, 마이그레이션, 트러블슈팅 관련 문서를 순차적으로 정리할 예정입니다.

## 문서 활용 방법

이 저장소는 아래와 같은 방식으로 활용할 수 있습니다.

1. 필요한 기술 영역의 디렉터리로 이동합니다.
2. 구축 또는 설정 문서를 순서대로 확인합니다.
3. 환경별 차이점은 별도 메모와 함께 비교 적용합니다.
4. 반복 수행 작업은 체크리스트나 스크립트로 다시 정리합니다.
5. 운영 중 발생한 이슈는 트러블슈팅 문서로 확장합니다.
