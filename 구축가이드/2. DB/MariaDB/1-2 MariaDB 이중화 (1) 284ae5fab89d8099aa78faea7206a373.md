## 3. 어플리케이션 중지 및 DB 재기동

- 두 서버 모두 어플리케이션 중단
- 각 서버에서 DB 서비스 재기동

```bash
systemctl restart mariadb
```

---

## 4. MariaDB 설정 수정

### Master 서버

```
# /etc/my.cnf.d/server.cnf
[mysqld]
log-bin = mysql-bin
server-id = 1
binlog_format = row
```

### Slave 서버

```
# /etc/my.cnf.d/server.cnf
[mysqld]
log-bin = mysql-bin
server-id = 2
binlog_format = row
```

---

## 5. Master DB Export

```bash
mysqldump -u root -p mcapi > mcapi.sql
```

> 전체 백업 시 `mysqldump --all-databases` 사용 가능.
> 

> 단, `mysql.user` 테이블까지 포함되므로 주의.
>

## 6. SQL 파일 전송 (Master → Slave)

```bash
sftp 계정명@<Slave_IP>
put mcapi.sql
bye
```

## 7. Slave DB Import

```sql
-- mysql 클라이언트에서 실행
source /경로/mcapi.sql;
```

## 8. Replication 계정 생성

### Slave 서버에서 실행 (Master IP 지정)

```sql
GRANT REPLICATION SLAVE ON *.* TO 'repli'@'172.16.0.149' IDENTIFIED BY 'repli';
FLUSH PRIVILEGES;
```

### Master 서버에서 실행 (Slave IP 지정)

```sql
GRANT REPLICATION SLAVE ON *.* TO 'repli'@'172.16.0.198' IDENTIFIED BY 'repli';
FLUSH PRIVILEGES;
```

## 9. Master 상태 확인

```sql
SHOW MASTER STATUS;
```

- binlog 파일명(File)과 위치(Position)를 확인

---

## 10. Master–Slave 연동 설정

각 서버에서 서로의 정보를 지정합니다.

```sql
CHANGE MASTER TO
  MASTER_HOST='상대방_IP',
  MASTER_USER='repli',
  MASTER_PASSWORD='repli',
  MASTER_LOG_FILE='mysql-bin.000001',
  MASTER_LOG_POS=1234;
```

Slave 기동

```sql
START SLAVE;
```

---

## 11. 상태 확인

```sql
SHOW SLAVE STATUS\G;
```

- Master_Host가 상대방 IP로 설정되어 있는지 확인
- Slave_IO_Running과 Slave_SQL_Running이 YES 인지 확인

---

## 12. Failover 및 복구

- 계획된 점검: 어플리케이션 쓰기 트래픽 정지 → Slave 최신화 확인 → 승격
- 비계획 장애: Master 불가 시, 최신 Slave 승격 후 다른 Slave는 새 Master로 재구성

승격 예시(간단)

```sql
STOP SLAVE;
RESET SLAVE ALL;
SET GLOBAL read_only=0;  -- SUPER 권한 계정으로
-- VIP/프록시 전환 또는 DNS 업데이트 수행
```

구 Master 복구 후 재참여

- 재설치 또는 데이터 동기화 후 새 Master를 기준으로 CHANGE MASTER 구성

---

## 13. 모니터링 포인트

- Seconds_Behind_Master 추이, IO/SQL 스레드 상태, Last_*_Error
- 디스크 용량(binlog, relay log), IOPS, 네트워크 지연

---

## 14. 자주 발생하는 에러와 대응

- 1045 Access denied: 계정/비밀번호/호스트 권한 재확인
- 2003 Can't connect: 방화벽·보안그룹·DNS·포트 점검
- 1236 Could not find first log file: Master에서 binlog 만료 또는 삭제. 재덤프 후 재구성
- Duplicate key on slave: 충돌 레코드 정리 또는 재동기화
