# 1-1 DB 내 스키마, 테이블에 적용 된 캐릭터셋 변경

<aside>
⚠️

이 문서는 운영 중인 MariaDB의 스키마·테이블·기본 설정 character set을 latin1에서 utf8로 통일하는 표준 절차입니다. 작업 전 전체 백업과 서비스 중단을 반드시 수행하세요.

</aside>

---

## 목표

- 모든 스키마, 테이블, 서버 기본 설정의 character set과 collation을 utf8, utf8_general_ci로 통일
- 변환 후 데이터 정합성 확인

## 사전 준비 및 주의사항

- DB와 연결된 모든 애플리케이션을 중단 후 진행
- 전체 데이터베이스 Full Backup 필수 보관
- 복구 시나리오 및 소요 시간 사전 점검

---

## 절차

### 1) 전체 백업 수행

```bash
mysqldump -u root -p --all-databases > /path/to/backup.sql
```

### 2) 서버 기본 문자셋 설정(my.cnf)

my.cnf에 다음을 추가 또는 수정합니다.

```
[mysqld]
skip-character-set-client-handshake
character-set-server = utf8
collation-server = utf8_general_ci
init-connect = 'SET NAMES utf8'
```

### 3) MariaDB 재기동 및 설정 확인

```bash
systemctl restart mariadb
```

```sql
SHOW VARIABLES LIKE 'c%';
```

### 4) 변환을 위한 스키마 정리(필요 시)

- 기존 스키마를 백업에서 재생성할 계획이라면, 대상 스키마를 정리합니다.
- 파괴적 작업이므로 백업 보관 상태를 재확인하세요.

```sql
-- 예시: 특정 스키마 제거
DROP DATABASE 스키마명;
```

### 5) 백업 파일 내 문자셋·콜레이션 일괄 치환

vi 등 에디터에서 명령 모드로 다음을 수행합니다.

```
:%s/latin1_swedish_ci/utf8_general_ci/g
:%s/latin1/utf8/g
```

- 목표: `CHARACTER SET utf8 COLLATE utf8_general_ci`

### 6) 데이터 복구(Import)

```sql
-- mysql 클라이언트에서 실행
source /path/to/backup.sql
```

### 7) 스키마 기본 문자셋 확인

```sql
SELECT
  schema_name AS 'Database',
  default_character_set_name AS 'Character Set',
  default_collation_name     AS 'Collation'
FROM information_schema.SCHEMATA;
```

### 8) 테이블 문자셋·콜레이션 확인

```sql
SELECT
  T.table_schema,
  T.table_name,
  CCSA.character_set_name AS 'character_set',
  T.table_collation
FROM information_schema.`TABLES` T
JOIN information_schema.`COLLATION_CHARACTER_SET_APPLICABILITY` CCSA
  ON CCSA.collation_name = T.table_collation
WHERE T.table_schema = '조회할_스키마_이름';
```
