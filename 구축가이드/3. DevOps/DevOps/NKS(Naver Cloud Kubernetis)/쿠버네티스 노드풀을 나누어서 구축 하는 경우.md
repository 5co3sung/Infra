# 쿠버네티스 노드풀을 나누어서 구축 하는 경우

1. 한 개의 쿠버네티스에서 노드풀로 구분 할 경우 헬름차트에서는 다음을 수행해야 한다.
- applications에서 신규 노드풀의 생기는 파드를 만들 경우, Path는 같게 하되 이름, values 파일, 네임스페이스는 다르게 하여야 한다.

예) Ingress NGINX

```yaml
- name: "ingress-nginx-controller" # 기존에 떠 있던 서비스와 다르게 설정
  path: "ingress-nginx"
  namespace: "ingress-nginx"
  project: "infra"
  valueFiles: "values-dev.yaml" # 기존에 적용된 파일 말고 다르게
  # syncOptions: []
  automated:
    enabled: false
```

- 어플리케이션 외 관리형 소프트웨어(인그레스 게이트웨이, 게이트웨이)의 헬름차트에서도 values 파일의 명칭을 별도로 구분하여 생성해주어야 한다.
1. 게이트웨이 부문
- 예를 들어 `nfh-values-dev.yaml` 파일을 하나 만들고 아래 `hosts` 부분은 사용할 도메인으로 바꾼다.

```yaml
fullnameOverride: nfh-gateway
hosts:
  - "nfh-dev.lemonhc.com"
tls:
  mode: SIMPLE
  credentialName: lemonhccom-2025
  minProtocolVersion: TLSV1_2
  maxProtocolVersion: TLSV1_3
```

1. 인그레스 게이트웨이 부분
- `annotations` 부분에서 `description` 부분만 변경한다. (개발 구성이라 서브넷은 같은 대역을 사용)

```yaml
annotations:
  # service.beta.kubernetes.io/ncloud-load-balancer-backend-protocol: "tcp"
  # service.beta.kubernetes.io/ncloud-load-balancer-proxy-protocol: "true"
  service.beta.kubernetes.io/ncloud-load-balancer-idle-timeout: "60"
  service.beta.kubernetes.io/ncloud-load-balancer-subnet-id: "48668"
  service.beta.kubernetes.io/ncloud-load-balancer-description: "소방병원 개발 K8S LB"
  service.beta.kubernetes.io/ncloud-load-balancer-size: "small"

# "_internal_defaults_do_not_set" is a workaround for Helm limitations.
# Users should NOT set "._internal_defaults_do_not_set" explicitly,
# but rather directly set the fields internally.
_internal_defaults_do_not_set:
  # Name allows overriding the release name. Generally this should not be set
  name: "nfh-istio-ingressgateway"
```

- `name` 부분은 applications에 적용한 이름을 넣어준다.
1. 어플리케이션 부분 (VirtualService 설정)
- `hosts`와 `gateways` 부분을 다른 것으로 변경

```yaml
virtualService:
  # hosts 생략 시 mesh로 설정되며 클러스터 내부에서만 virtualService가 적용된다.
  # hosts: []
  hosts:
    - "nfh-dev.lemonhc.com"
  gateways:
    - istio-system/nfh-gateway
```

- 노드풀 affinity 설정 (노드풀 생성 시 설정한 key/value 값 사용)

```yaml
affinity:
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
        - matchExpressions:
            - key: nfh
              operator: In
              values:
                - "true"
```

추가

- 가끔씩 헬름차트에서 배포한게 적용이 안되는 경우가 있다. 이럴 경우 ArgoCD의 Redis(캐시)가 지워지지 않는 경우가 간혹 있다고 하니 ArgoCD의 Redis 파드를 삭제 후 재생성한다.
