# database_backup_retention_notes

## 1. 개요
정기 백업과 보관 주기 관리를 스크립트 기반으로 운영할 때의 핵심 포인트를 정리했습니다.

## 2. 작업 배경
백업은 생성 자체보다 **보관 주기, 삭제 기준, 복구 가능성 검증**이 중요합니다.
단순 덤프 스크립트만 두면 어느 시점 파일을 남기고 지울지 불명확해질 수 있습니다.

## 3. 문제/요구사항
- 일 단위 백업 파일을 순환 보관해야 함
- 과도한 저장 공간 사용을 막아야 함
- 백업 성공 여부와 복구 가능성을 주기적으로 확인해야 함

## 4. 판단 및 선택 이유
- 날짜별 디렉터리 순환 방식은 이해가 쉽고 운영이 단순함
- 다만 삭제/이동 순서가 잘못되면 보관 체인이 깨질 수 있어 검증이 필요
- 백업 스크립트 문서에는 복구 테스트 절차까지 포함하는 것이 좋음

## 5. 적용 절차 요약
- cron 등록
- 날짜별 디렉터리 이동/삭제
- `mysqldump` 또는 DB별 백업 명령 수행
- 권한 부여
- 복구 테스트 주기 수립

## 6. 검증 방법
- 백업 파일 생성 여부
- 날짜별 디렉터리 순환 상태
- 샘플 복구 테스트
- 용량 및 보관일수 확인
