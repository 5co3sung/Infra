# db_migration_runbook

## 1. 개요
DB 분리 및 이관 작업을 수행할 때 필요한 최소 절차를 런북 형태로 정리한 문서입니다.

## 2. 작업 배경
DB 이관은 단순 데이터 복사보다 서비스 중지 시점, 백업 시점, 스키마 사전 준비, 검증 절차가 더 중요합니다.
이 과정이 문서화되어 있지 않으면 다운타임과 복구 리스크가 커집니다.

## 3. 문제/요구사항
- 서비스 공지와 작업 창구 정리가 먼저 필요
- 애플리케이션 중지와 백업 시점이 맞아야 함
- 스키마/DB 생성 같은 사전작업이 누락되면 이관 중 장애가 발생할 수 있음

## 4. 판단 및 선택 이유
- 이관 문서는 명령어 모음보다 **작업 순서**가 핵심
- 사전 공지 → 서비스 정지 → 백업 → 이관 → 검증 → 재오픈 흐름으로 작성하는 편이 실무적임
- 롤백 기준이 포함되어야 운영 문서로서 가치가 있음

## 5. 적용 절차
1. 서비스 공지 및 작업 승인
2. 애플리케이션 정지
3. 전체 또는 대상 DB 백업
4. 대상 DB/스키마/계정 사전 생성
5. 데이터 이관 수행
6. 애플리케이션 연결 정보 변경
7. 기능 검증 및 로그 확인
8. 서비스 재오픈
9. 백업본 보관 및 작업 결과 기록

## 6. 검증 방법
- 테이블 수/행 수 비교
- 주요 쿼리 실행 여부 확인
- 애플리케이션 연결 테스트
- 오류 로그 및 문자셋 깨짐 여부 확인

## 7. 결과 및 운영 효과
- 이관 작업이 개인 경험 의존형이 아니라 재사용 가능한 절차 문서가 됨
- 작업 누락 가능성을 줄일 수 있음

## 8. 운영 체크포인트
- 백업 없이 바로 이관하지 않기
- 애플리케이션 중지 범위를 명확히 정의
- 이관 후 성능, 권한, 문자셋, 트리거/이벤트 동작까지 확인
