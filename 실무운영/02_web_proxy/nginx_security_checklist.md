# nginx_security_checklist

## 1. 개요
Nginx 운영 시 반복적으로 확인하게 되는 취약점 점검 항목과 보안 가이드를 포트폴리오형 체크리스트로 정리했습니다.

## 2. 작업 배경
서비스 오픈이나 ISMS/내부 보안 점검 전에는 기능 구현보다
프로세스 권한, 디렉터리 리스팅, 업로드 용량 제한, 로그 보관, 패치 상태 같은 운영 항목을 먼저 확인해야 했습니다.

## 3. 주요 점검 항목
- 웹 서비스 영역 분리
- 불필요 파일/매뉴얼 제거
- 심볼릭 링크 사용 제한
- 업로드 파일 크기 제한
- 디렉터리 리스팅 비활성화
- 웹 프로세스 권한 최소화
- 로그 포맷 및 보관 기간 확인
- 안정화 버전/패치 상태 확인

## 4. 판단 및 선택 이유
- 웹 서버는 가장 먼저 노출되는 계층이므로 애플리케이션보다 먼저 기본 보안면을 정리해야 함
- 보안 점검은 기능보다 운영 기준의 일관성을 요구하므로, 서버마다 같은 체크리스트가 필요함
- 단순 설정값 나열보다 '왜 필요한가'를 같이 기록해야 재사용성이 높아짐

## 5. 검증 방법 예시
- `nginx.conf`, `conf.d/*.conf`에서 `autoindex`, `client_max_body_size`, `disable_symlinks` 확인
- 프로세스 계정 및 권한 확인
- `curl -I`, 디렉터리 접근 테스트
- 로그 경로 및 로테이션/보관 정책 확인
- 버전 조회 및 패치 이력 확인

## 6. 운영 효과
- 점검 대응 속도가 빨라짐
- 신규 서버에도 동일한 기준을 적용하기 쉬움
- '설치 문서'가 아닌 '운영 기준 문서' 역할을 할 수 있음
