# ngrinder_installation_and_usage

## 1. 개요
NGrinder Controller/Agent 설치와 기본 사용 흐름을 성능 테스트 관점에서 정리한 문서입니다.

## 2. 작업 배경
애플리케이션 성능이나 Pod 수에 따른 TPS 변화를 보기 위해,
테스트 도구를 단순 설치하는 수준이 아니라 Controller-Agent 구조로 반복 사용 가능한 형태로 준비할 필요가 있었습니다.

## 3. 문제/요구사항
- NGrinder는 특정 Java 버전 요구사항이 있음
- Controller와 Agent 간 통신 포트/호스트를 정확히 맞춰야 함
- 테스트 결과를 단순 성공/실패가 아니라 성능 지표로 읽어야 함

## 4. 판단 및 선택 이유
- 부하 테스트는 운영 전 병목을 미리 찾는 데 유효함
- Agent 분리형 구조가 실제 분산 부하를 재현하기 좋음
- 단순 설치보다 **테스트 시나리오와 결과 해석**이 포트폴리오에 더 중요함

## 5. 적용 절차
1. Controller용 디렉터리 준비
2. 요구 Java 버전 설치
3. Controller WAR 실행
4. 웹 UI 접속 및 Agent 패키지 다운로드
5. Agent 서버에 업로드 및 설정
6. Agent 기동
7. 시나리오 작성 후 테스트 수행

## 6. 검증 방법
- Controller UI 접속
- Agent 연결 상태 확인
- 샘플 테스트 실행
- TPS, 응답시간, 에러율 수집

## 7. 결과 및 운영 효과
- 성능 테스트 환경을 반복적으로 재사용할 수 있음
- 향후 'Pod 수 변화에 따른 성능 변화' 같은 실험 문서로 확장 가능

## 8. 운영 체크포인트
- Java 버전 호환성 확인
- Controller는 공인 IP보다 내부 통신 기준 주소를 확인
- 테스트 포트 범위와 방화벽 상태 확인
