# yara_installation_notes

## 1. 개요
YARA를 이용해 악성코드 탐지 룰을 적용할 때 필요한 설치 메모와 운영 관점을 정리했습니다.

## 2. 작업 배경
보안 이벤트 대응이나 점검 과정에서 IOC 기반 점검만으로는 한계가 있어,
룰 기반으로 파일/패턴을 탐지할 수 있는 도구가 필요했습니다.

## 3. 판단 및 선택 이유
- YARA는 룰 기반 탐지가 가능해 특정 악성코드 계열 점검에 적합함
- 패키지 설치가 가능하면 빠르게 도입할 수 있고, 필요 시 소스 빌드도 가능함
- 보안 도구 설치 자체보다 **룰 관리와 점검 절차**가 더 중요함

## 4. 적용 절차 요약
- 패키지 또는 소스 기반으로 YARA 설치
- 필요한 컴파일 도구 설치
- 룰 파일 작성 또는 외부 제공 룰 확보
- 테스트 파일/운영 경로 대상으로 점검 수행

## 5. 검증 방법
- `yara --version`
- 샘플 룰 로딩 여부 확인
- 지정 경로 스캔 결과 확인

## 6. 운영 체크포인트
- 공개된 룰을 바로 쓰기보다 오탐 가능성을 먼저 검토
- 운영 서버에서는 전체 파일시스템 스캔 범위와 시점을 조정
- 룰 변경 이력과 적용 기준을 문서화
