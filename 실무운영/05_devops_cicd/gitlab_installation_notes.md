# gitlab_installation_notes

## 1. 개요
GitLab CE를 자체 서버에 설치할 때 필요한 패키지, 외부 URL, 메일 연동 고려사항을 정리한 문서입니다.

## 2. 작업 배경
형상관리와 협업 도구를 내부 환경에 직접 구축해야 하는 경우,
설치 자체보다 외부 접근 URL, 메일 발송, 의존성 패키지 호환성 검토가 중요했습니다.

## 3. 문제/요구사항
- 패키지 의존성 이름이 OS 버전에 따라 달라질 수 있음
- `EXTERNAL_URL` 값을 잘못 주면 접속 경로가 꼬일 수 있음
- 설치 후 메일, 사용자 생성, Runner 연동 등 후속 작업이 필요함

## 4. 판단 및 선택 이유
- 설치 문서에는 단순 명령보다 운영에 직접 영향을 주는 URL/포트/인증 메일 항목을 앞에 두는 것이 좋음
- 자체 GitLab은 편하지만 패치, 백업, 저장소 용량 관리 책임도 같이 가져가야 함

## 5. 적용 절차 요약
- 의존성 패키지 설치
- Postfix 등 메일 관련 패키지 검토
- GitLab 공식 저장소 등록
- `EXTERNAL_URL` 지정 후 설치
- 웹 UI 접속 및 기본 설정 수행

## 6. 검증 방법
- GitLab UI 접속
- 프로젝트 생성 테스트
- 메일 발송 테스트
- 서비스 상태 확인

## 7. 운영 체크포인트
- 공인/사설 URL 기준을 명확히 둘 것
- 대용량 로그/저장소 백업 전략 필요
- Runner 분리 여부와 SSL 적용 여부를 함께 검토
