# jenkins_bootstrap_and_operations

## 1. 개요
Jenkins 신규 설치와 초기 관리자 설정, Java 버전 호환성, 운영 중 주의사항을 정리한 문서입니다.

## 2. 작업 배경
배포 자동화나 빌드 표준화를 위해 Jenkins를 도입할 때,
설치 명령보다 먼저 Java 버전/환경변수/Repository GPG 이슈를 해결해야 했습니다.

## 3. 문제/요구사항
- Jenkins 버전에 따라 요구 Java 버전이 달라짐
- `JAVA_HOME` 또는 대체 버전 선택이 맞지 않으면 서비스가 기동하지 않음
- 레포 GPG Key 만료/검증 오류가 발생할 수 있음

## 4. 판단 및 선택 이유
- 설치 전 Java 기준 버전을 먼저 정해야 함
- 단일 서버에 Java가 여러 버전일 수 있어 `alternatives` 관리가 중요함
- 설치 완료 후에도 불필요한 플러그인을 최소화하는 것이 운영에 유리함

## 5. 적용 절차
1. 지원 Java 버전 확인 및 설치
2. `JAVA_HOME` 설정
3. Jenkins 저장소 등록
4. GPG Key 문제 시 대체 키/레포 확인
5. Jenkins 설치 및 기동
6. 초기 관리자 비밀번호 확인
7. 필요한 플러그인만 선택 설치
8. 관리자 계정 생성 및 기본 환경 설정

## 6. 검증 방법
- `java -version`
- `systemctl status jenkins`
- 브라우저 접속 및 초기 Unlock 화면 확인
- 빌드 실행 테스트

## 7. 결과 및 운영 효과
- 설치 과정에서 흔한 Java/GPG 이슈를 문서화할 수 있음
- 단순 설치 문서를 넘어 운영성 있는 Jenkins 구축 메모로 전환 가능

## 8. 운영 체크포인트
- Java 버전 다중 설치 시 실제 사용 중인 경로를 명확히 관리
- 기본 플러그인 일괄 설치보다 필요한 항목만 선별
- Jenkins 홈 디렉터리 백업과 플러그인 업데이트 정책을 함께 관리
