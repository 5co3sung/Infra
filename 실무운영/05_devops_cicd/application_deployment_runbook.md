# application_deployment_runbook

## 1. 개요
애플리케이션 배포 작업을 수동으로 수행할 때의 절차를 운영 런북 형태로 정리한 문서입니다.

## 2. 작업 배경
현장 배포는 단순 파일 전송보다,
배포 대상 확인, 서비스 중지/기동, 로그 확인, 기능 검증, 커뮤니케이션까지 포함한 절차 관리가 중요했습니다.

## 3. 문제/요구사항
- 배포 경로와 서버 경유 경로가 복잡할 수 있음
- 작업 전/후 커뮤니케이션이 없으면 운영 혼선이 생김
- 기동 성공과 기능 정상 동작은 별개의 확인 항목임

## 4. 판단 및 선택 이유
- 배포 절차는 명령어보다 순서가 중요
- 전송 → 백업/교체 → 중지 → 기동 → 로그 확인 → 기능 확인 흐름으로 정리해야 재사용 가능
- 배포 완료 보고 기준까지 문서에 포함해야 운영 문서로서 완성도가 높음

## 5. 적용 절차
1. 배포 대상/버전 확인
2. 파일 수령 및 대상 서버 전송
3. 서버 내 배포 경로 확인
4. 기존 파일 백업 또는 이름 변경
5. 공지 후 서비스 중지
6. 신규 버전 기동
7. 로그 모니터링
8. 기능/API 검증
9. 버전 정보 확인 및 결과 공유

## 6. 검증 방법
- 프로세스 상태 체크
- 애플리케이션 로그 확인
- 핵심 API 테스트
- 버전 문자열 확인

## 7. 결과 및 운영 효과
- 수동 배포라도 누락 없는 표준 절차를 유지할 수 있음
- 향후 Jenkins/ArgoCD 등 자동화 도입 전 기준 런북으로 활용 가능

## 8. 운영 체크포인트
- 배포 전 기존 버전 백업 필수
- 배포 시작/완료 커뮤니케이션 기준 명확화
- 기능 테스트 없이 '기동 성공'만으로 완료 처리하지 않기
